# 一、读者、写者问题与读写锁

## 1.1 读者、写者问题

这里的读者、写者指的是线程。读者、写者问题，发生在读的需求大、写的需求小的情况下

### 1.1.1 与生产者、消费者的区别

都是对于临界区资源访问的情况，但是实际上有核心上的不同：读者写者问题中，读者取完数据后，数据还是原来的数据，没有发生改变；生产者消费者中，消费者是会“取走”数据，使得其他的消费者无法再取到原来的数据

### 1.1.2 321 原则

**三种关系**

1. 写者和写者之间：互斥。一个线程在写的时候其他线程不能写，否则数据错乱
2. 读者和读者之间：并发（或者说没有关系也可以，因为一个线程读不会对其他线程造成影响）
3. 读者和写者之间：互斥、同步。互斥是因为写者在写的过程中，如果读者去读数据，可能会读到“脏数据”；同步是为了告知当前已经有数据写入了，可以被读取（就像家长做好饭会喊你吃饭）

**两个角色**

1. 写者：负责生产数据的线程
2. 读者：负责读取数据的线程

**一个共享区**

即一块缓冲区，用来存放数据

### 1.1.3 读者优先和写者优先

**读者优先**，指的是在有线程想要写时，已经有线程在读，此时也有新的线程还想要读，那么就让这个新来的读者去读，写者等待。默认采用的策略也是读者优先。

那如果一直都有新的读者呢？不就一直都写不了数据了吗？其实是不会的。

一方面，不是一直都会有读者，就像很火的视频凌晨也不一定有很多人看；另一方面，在实际项目中，临界区代码只占总体代码的一小部分，更多的是获取到临界资源后的处理。

**写者优先**，就是反过来，优先让写者去写数据，读者等待。如果平衡不好可能会导致**读者饥饿**。

## 1.2 读写锁

Linux 的 `pthread` 库中就为我们提供了读写锁。

### 1.2.1 锁的初始化与释放

```cpp
 int pthread_rwlock_destroy(pthread_rwlock_t *rwlock);
 int pthread_rwlock_init(pthread_rwlock_t *restrict rwlock,
     const pthread_rwlockattr_t *restrict attr);
 pthread_rwlock_t rwlock = PTHREAD_RWLOCK_INITIALIZER;
```

### 1.2.2 读者上锁

```cpp
int pthread_rwlock_rdlock(pthread_rwlock_t *rwlock);
int pthread_rwlock_tryrdlock(pthread_rwlock_t *rwlock);
```

### 1.2.3 写者上锁

```cpp
int pthread_rwlock_wrlock(pthread_rwlock_t *rwlock);
```
